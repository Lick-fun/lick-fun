# Session — 2026-07-16 — Profile Activity tabs: token names + USD values

## Task
User request: "in user profiles, i would like the holdings to show the tokens name
instead of contract address. I would like all activity on the profiles pages to show
the name of the token with usd value traded."

Continuation of the 2026-07-16 profile/reputation upgrades session. That session
fixed token-name resolution on the Holdings + Tokens Created LISTS. This session
fixes the same blank-name issue on the Activity tabs AND adds USD value display
to all activity rows.

## Root causes (verified in code)
1. **HoldingsList showed address-like strings** — `useTokenHoldings.ts` set
   `name: tokenData.name || agg.name || \`${tokenId.slice(0, 6)}...\``. When the
   Envio indexer stored an empty `name`, this placeholder was a NON-empty string,
   so the `useTokensMeta` on-chain multicall in `page.tsx` (which only resolves
   when `!h.name`) was SKIPPED — the truncated-address placeholder leaked to UI.
2. **ActivityTabs TradeRow** — showed only `trade.token?.symbol ?? "???"` and a
   MON amount. No USD value, no token name. `monUsdPrice` was not passed in. The
   nested `trade.token` join was frequently empty (indexer blank name/symbol).
3. **Creates tab** — showed `token.symbol` as the action label and `token.name`
   on the right; no USD value.

## What was implemented (tsc clean, lint clean, type-check passed)

### Phase 1 — Fix holdings name fallback (useTokenHoldings.ts)
- Changed the `name`/`symbol` fallback in the `holdings` useMemo from truncated-
  address placeholder to EMPTY STRING: `name: tokenData.name || agg.name || ""`,
  `symbol: tokenData.symbol || agg.symbol || ""`.
- This lets the existing `useTokensMeta` on-chain multicall in `page.tsx` actually
  fire and fill real names via `name()`/`symbol()`. No other change needed —
  `page.tsx` already calls `useTokensMeta(holdingsForResolution)` and merges into
  `holdingsResolved`, which `HoldingsList` receives.

### Phase 2 — Upgrade ActivityTabs TradeRow (All / Buys / Sells)
- Added `monUsdPrice?: number | null` to `ActivityTabsProps`; threaded through
  `ActivityTabs` → `ActivityList` → `TradeRow`.
- In `page.tsx`: pass `monUsdPrice={monUsdPrice}` to `<ActivityTabs />`.
- In `page.tsx`: built `tradesResolved` — dedupes trades by `token_id`, runs them
  through `useTokensMeta` to resolve blank name/symbol via on-chain
  `name()`/`symbol()`, then merges resolved values back into each trade's nested
  `token` field. Passes `trades={tradesResolved}` to `ActivityTabs`.
- Rewrote `TradeRow` in `ActivityTabs.tsx`:
  - Primary line: `Buy {name}` / `Sell {name}` (truncate long names with
    `truncate` class).
  - Secondary line: `{symbol} · {timeAgo}`.
  - Right side primary: USD value (`$X.XX` = `monAmount * monUsdPrice`); secondary:
    `{formatMon(monAmount)} MON`. When `monUsdPrice` is null/0, shows MON as
    primary and omits USD (mirrors HoldingsList's `hasUsdPrice` guard).
- Added `formatUsd` helper (adaptive precision: K/M suffixes, extra precision for
  small fractional amounts).

### Phase 3 — Upgrade Creates tab
- In `page.tsx`: pass `tokensResolved` (already on-chain-resolved via
  `useTokensMeta`) instead of raw `tokens` as `createdTokens` — ensures names
  populated and `price` present.
- In `ActivityTabs.tsx` `CreatedActivityList`: added `monUsdPrice` prop; computes
  `mcUsd` via `formatMarketCapUsd(token.price.marketCapMon, monUsdPrice)`.
  Display: primary `Created {name}`; secondary `{symbol} · {timeAgo}`; right side
  market cap USD with `MC` label below.
